## 2025-04-07 - Avoid Redundant DB Initialization in SessionStore
**Learning:** Instantiating `SessionStore` repeatedly triggered `_init_db()` which ran `conn.executescript()` to enforce schemas. While SQLite is fast, doing this O(n) times is wasteful disk I/O. Using a module-level boolean cache initially broke in-memory testing (`":memory:"`) because in-memory databases require schema initialization per-connection.
**Action:** Used a `set` to track initialized database paths (e.g., `_initialized_paths = set()`) and explicitly bypassed the cache for `":memory:"` paths to ensure test isolation is preserved while preventing redundant disk I/O for physical files.
